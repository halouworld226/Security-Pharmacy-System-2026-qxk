## 快速部署

### 方式一：Docker Compose（最推荐，5 分钟启动）

```bash
# 1. 克隆项目
git clone https://github.com/halouworld226/Security-Pharmacy-System.git
cd Security-Pharmacy-System

# 2. 启动所有服务（数据库 + Web 应用）
docker-compose up -d

# 3. 等待数据库初始化完成（约 30-60 秒）

# 4. 浏览器访问 http://localhost:5000
```

### 方式二：Windows 一键完整部署

双击 `setup.bat` → 输入 `1` → 回车，自动完成：

```
部署数据库（Docker openGauss）
    ↓
数据库安全加固（SSL/权限/审计/加密...）
    ↓
Windows 防火墙配置（端口 15432 IP 白名单）
    ↓
备份配置说明
```

### 方式三：分步部署

```
setup.bat → [2] 仅部署数据库       # 第一步：部署数据库
setup.bat → [3] 部署 + 安全加固     # 部署并立即加固
setup.bat → [4] 仅运行安全加固     # 数据库已存在时单独加固
```

### 方式四：命令行部署

```bash
bash deploy.sh                      # 部署数据库
bash security_hardening.sh          # 安全加固
python app.py                       # 启动应用
```

## 安全加固

### 加固内容（共 7 项）

| # | 项目 | 说明 |
|---|------|------|
| 1 | **网络隔离** | 创建 Docker 内部网络，pg_hba.conf 配置 IP 白名单，仅允许 SSL 连接 |
| 2 | **SSL/TLS 传输加密** | 自建 CA 签发服务端证书，强制 SSL 加密传输 |
| 3 | **最小权限账号** | 撤销 druguser 超级权限，创建 app_rw / audit_readonly / backup_role 等角色 |
| 4 | **数据静态加密** | 创建加密密钥表，users.phone_encrypted 字段存储 AES 加密手机号 |
| 5 | **原生安全加固** | 删除匿名账号/测试库，收紧 CREATE 权限，配置密码策略 |
| 6 | **数据库原生审计** | 开启审计日志，记录登录/DDL/DML/权限变更等操作 |
| 7 | **SQL 注入防护** | 创建 SQL 注入监控视图，行级安全（RLS）实现门店数据隔离 |

### 加固后新增账号

| 用户名 | 密码 | 用途 |
|--------|------|------|
| `app_user` | `App@Secure2024!` | 业务应用连接数据库 |
| `auditor` | `Audit@2024!` | 审计人员只读查询 |
| `backup_op` | `Backup@2024!` | 备份操作专用 |

### 重新运行

加固脚本是幂等的，可重复执行：

```bash
bash security_hardening.sh
```

## 备份与容灾

使用 `backup_dr.sh` 进行加密备份。

### 常用命令

```bash
bash backup_dr.sh backup          # 执行加密备份（GPG AES256）
bash backup_dr.sh restore         # 容灾恢复演练（创建临时库验证）
bash backup_dr.sh list            # 查看备份文件列表
bash backup_dr.sh verify          # 验证最近备份完整性
```

### 备份说明

- 备份文件位于 `/d/backups/drugstore-db/`
- 使用 GPG 对称加密（AES256）
- 自动轮转：日备份保留 90 天，周备份保留 12 周
- 支持异地备份（修改 backup_dr.sh 中 BACKUP_ROOT 为网络路径）
- 也可通过 `setup.bat` 菜单选项 6/7/8 操作

## 防火墙配置

```bash
setup.bat → [5] 配置 Windows 防火墙
```

或命令行：

```powershell
powershell -ExecutionPolicy Bypass -File windows_firewall.ps1
```

放行规则：

| 来源 | 端口 | 策略 |
|------|------|------|
| 127.0.0.1（本机） | 15432 | 允许 |
| 192.168.0.0/16（内网） | 15432 | 允许 |
| 172.16.0.0/12（Docker） | 15432 | 允许 |
| 其他 | 15432 | 阻止 |

> 需要管理员权限运行。

## 常见问题

### 容器启动后闪退

```bash
docker logs opengauss    # 查看错误日志
docker rm -f opengauss   # 删除容器后重新部署
bash deploy.sh
# 或使用 docker-compose
docker-compose down && docker-compose up -d
```

### setup.bat 闪退

确保文件编码为 CRLF（Windows 格式）。用以下命令修复：

```bash
sed -i 's/$/\r/' setup.bat
```

### 脚本报错 "No such file or directory"

MSYS2 路径转换导致，脚本已内置 `export MSYS2_ARG_CONV_EXCL="*"` 处理此问题。

### 加固后容器无法启动

使用 `gs_guc` 的参数已做幂等处理，不会产生重复配置。如果仍出现问题，可重建容器后重新执行加固。

### 端口 15432 被占用

修改 `docker-compose.yml` 中的端口映射，例如改为 `15433:5432`，并同步修改 `.env` 中的 `DB_PORT=15433`。

### 首次启动后图表没有数据显示

系统内置的示例数据截止至 **2026年6月**。如遇到图表无数据展示，只需在系统中录入销售数据即可实时生成可视化图表。

### 如何查看应用日志

```bash
# 查看 Web 应用日志
docker logs pharmacy-web

# 查看数据库日志
docker logs opengauss
```
